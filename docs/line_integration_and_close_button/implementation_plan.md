# 実装計画 - 重複登録問題の修正

## 現状の分析
現在、診断を完了すると以下の2つのアクションがほぼ同時に発生し、両方がスプレッドシートに記録されています。

1.  **LIFFからの直接POST**: 診断結果の全データが含まれる。
2.  **LINE Webhook経由のPOST**: LIFFがトークに送信したメッセージに反応して発生する。内容が単なる「メッセージ」であるため、診断項目（Q1〜Q4、タイプ）が空になる。

## 修正内容

### gas/Code.gs
- `saveToSheet` 関数において、`data.resultType` が存在しない場合は、診断結果の記録として不完全であるため、スプレッドシートへの追記をスキップするようにします。
- これにより、LIFFからのメッセージ送信に付随して発生する Webhook イベント（診断項目を含まない単純なメッセージイベント）が記録されなくなります。

## 検証計画
- 修正後、診断を1回実行し、スプレッドシートに1行だけ（全ての値が埋まった状態）記録されることを確認。
- LINEで普通のメッセージを送ってみて、スプレッドシートに空の行が追加されないことを確認。
