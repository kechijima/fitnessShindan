# 実装計画 - スプレッドシート保存項目の不備修正

## 現状の分析
現在、スプレッドシートに新しい行は追加されるものの、日時とフォロー状況以外の項目が空になっています。これは、GASの `Code.gs` 側で受信した `e.postData.contents` が空であるか、正しくパースできていないことを示唆しています。

### 推定される原因
- **CORSとfetchの制限**: `fetch` で `mode: 'no-cors'` かつ `Content-Type: application/json` を指定すると、ブラウザがリクエストボディを正しく送信しない、あるいはヘッダーが削除される場合があります（GASがJSONとして認識できない）。
- **ペイロードの欠損**: LIFF SDKからのプロフィール取得などが間に合っていない場合、プロパティが `undefined` になり、`JSON.stringify` で項目が消えてしまう可能性があります。

## 修正内容

### index.html
- `fetch` の `Content-Type` を `text/plain` に変更し、確実にリクエストボディが GAS に届くようにします。
- `submitDiagnosis` 内でペイロードの内容をブラウザのコンソールに表示し、送信前にデータが揃っているか確認できるようにします。
- プロパティが `undefined` の場合に備え、デフォルト値を設定します。

### gas/Code.gs
- `doPost` 内で、JSONパースが失敗した場合や、期待するキーが不足していた場合に実行ログにエラーを出力するように強化します。

## 検証計画
- 修正後、ブラウザの開発者ツール（コンソール）を開き、`[DEBUG] payload:` に正しいデータが表示されているか確認。
- 再度診断を実行し、スプレッドシートに全項目が反映されるか確認。
